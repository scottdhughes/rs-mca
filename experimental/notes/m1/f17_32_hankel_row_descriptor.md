# F17^32 Hankel Row Descriptor

Status: AUDIT.

The M3 regular-window program needs a concrete row descriptor for

```text
C = RS[F_17^32,H,256],    |H| = 512.
```

The descriptor is generated by

```text
experimental/scripts/emit_f17_32_hankel_row_descriptor.py
```

and written to

```text
experimental/data/certificates/hankel-f17-32-row-descriptor/
  f17_32_n512_k256_hankel_row_descriptor.json
```

It pins the polynomial-basis field model, using the same degree-32 modulus as
`experimental/scripts/verify_v1_f17_32_algebra_checker.py`, converted to the
low-degree-first encoding used by the regular-minor extractor and packet
checker.

The emitted domain is the order-512 multiplicative subgroup generated by the
encoded element

```text
1415472521070614055149723472223960876430.
```

The resulting domain hash is

```text
35904a892e0319b3805e91438ec2733427a351a72ce9654428d6a33bd3575b92.
```

The descriptor checks:

```text
modulus is monic degree 32 and irreducible over F_17;
v2(17^32 - 1) = 9;
the generator has exact order 512;
the domain has 512 distinct nonzero elements and contains 1;
floor(17^32 / 2^128) = 6;
the M3 regular non-tangent window is 385 <= A <= 426;
the regular degree-bound sum over that window is 4515.
```

This removes an ambiguity from future M3 packets: `row.domain_hash` and the
polynomial-basis field model are now fixed before any syndrome-pencil line data
are supplied.

Non-claims: the descriptor does not supply a line, does not compute any regular
minor, and does not prove a safe-side MCA bound.
