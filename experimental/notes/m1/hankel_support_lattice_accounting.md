# Hankel Support-Lattice Accounting

Status: PROVED.

Source DAG node: `f_support_lattice`.

Depends on: `f_dual_distance_frame`.

## Statement

The naive sparse-dual descent tree can branch exponentially if every sparse
support is treated as a new binary choice. The correct accounting object is the
closed-set lattice generated by sparse-dual supports. Tree nodes with the same
closed set, residual degree, and flat data have the same future recursion, so
memoization identifies the binary descent tree with a directed walk on this
support lattice.

Along every strict chain, either the flat dimension drops or the residual
locator degree drops. Thus the chain length is bounded by the initial dimension
plus the initial degree-drop budget, and the memoized descent size is bounded by
the number of closed sets in the generated support lattice, up to the recorded
bounded weight-label factor.

## Proof

By `f_dual_distance_frame`, sparse trace degeneracies are exactly sparse dual
supports. A descent step indexed by such a support replaces the current state by
subproblems determined by the closed sets generated from those supports:
unions, trace closures, corresponding subflats, and residual domains.

If two branch histories reach the same closed set with the same residual degree
and flat data, all subsequent decisions see the same available sparse-dual
supports and the same residual object. The future recursive calls are therefore
identical. Memoizing by this closed state collapses repeated binary histories
to one lattice state.

A strict transition cannot preserve both flat dimension and residual degree. If
it did, the branch would have reached the same closed state and would be
memoized. Hence any strict chain has length at most the available dimension
drops plus the available degree drops. The total number of memoized states is
bounded by the number of closed sets in the generated support lattice, with the
bounded label factor recorded by the descent convention.

## Non-Claims

This packet is an accounting identity. It does not prove that the closed-set
lattice is polynomial for every family. The family-specific polynomial bound is
supplied later for the Hankel rank-profile setting.
