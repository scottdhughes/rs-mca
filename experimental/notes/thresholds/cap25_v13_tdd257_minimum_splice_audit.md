# CAP25 v13 TDD257 minimum-splice audit

Status: PROVED / AUDIT / EXACT_NEW_WALL.

This note is a follow-up to
`cap25_v13_seven_slope_ccl_tdd_split.md`.  It records what the first
minimum-splice audit of the `TDD257` branch actually proves.  It is not a
CAP25 closure and does not claim that `TDD257` is paid by `CCL_149` or by the
all-seven common-GCD branch alone.

## Setup

Use the deployed row and notation from the seven-slope split note:

```text
F = F_{17^32},   H = mu_512,   C = RS[F,H,256],
n = 512,         k = 256,      d(C) = 257.
```

At agreement `A = 384`, write `j = n-A = 128`.  For seven distinct finite
retained slopes,

```text
y_i = f_0 + gamma_i f_1 = c_i + e_i,
c_i in C,
E_i = supp(e_i),
|E_i| = 128.
```

For a triple `{i,j,k}`, set

```text
Delta_ijk =
  (gamma_j - gamma_k)c_i
  + (gamma_k - gamma_i)c_j
  + (gamma_i - gamma_j)c_k.
```

Then `Delta_ijk in C`, and the same affine combination of the received line
gives

```text
supp(Delta_ijk) subset E_i union E_j union E_k.
```

Throughout this note, for an active triple put

```text
U = E_i union E_j union E_k,
M_U(X) = prod_{h in H \ U}(X-h).
```

## TDD factorization

If `Delta = Delta_ijk` is active, then `Delta` vanishes on `H \ U`, hence

```text
Delta = M_U R
```

for some polynomial `R`.  Since `Delta` is represented by a polynomial of
degree `< 256`,

```text
deg R <= 255 - (512 - |U|) = |U| - 257.
```

Thus `|U| >= 257` for any active triple.  The hidden factor `R` is essential:
one may replace `Delta = M_U R` by `Delta = lambda M_U` only in the exact
`|U| = 257` case, or after separately accounting for all roots of `R`.

## Minimum defect and the silent shell

Assume now that `Delta` has minimum RS weight:

```text
wt(Delta) = 257.
```

Let

```text
S = supp(Delta),   Z = U \ S.
```

Then `Delta` has exactly `255` zeros on `H`, so

```text
Delta =
  lambda
  (prod_{h in H \ U}(X-h))
  (prod_{z in Z}(X-z))
```

for some `lambda != 0`.

Every point of `Z` belongs to the triple union but has zero defect value.  It
cannot lie in exactly one of `E_i,E_j,E_k`, since the corresponding coefficient
`gamma_j-gamma_k`, `gamma_k-gamma_i`, or `gamma_i-gamma_j` is nonzero and a
single nonzero error value cannot cancel itself.  Therefore each point of `Z`
has at least two memberships among the three error supports.

The total membership count is

```text
|E_i| + |E_j| + |E_k| = 384.
```

The `257` points of `S` use at least one membership each, and the points of
`Z` use at least two memberships each.  Hence

```text
384 >= 257 + 2|Z|,
```

so

```text
|Z| <= 63.
```

Equivalently, in the minimum-defect branch the hidden factor `R` is a locator
for a silent shell of size at most `63`.

## Pair-common consequences

Let

```text
P =
  |E_i cap E_j|
  + |E_i cap E_k|
  + |E_j cap E_k|,
T = |E_i cap E_j cap E_k|.
```

Inclusion-exclusion gives

```text
|U| = 384 - P + T.
```

In the minimum-defect case, `|U| = |S| + |Z| = 257 + |Z|`, so

```text
P - T = 127 - |Z|.
```

Since `|Z| <= 63`, one has `P >= 64`.  Therefore some pair among the active
triple satisfies

```text
|E_a cap E_b| >= ceil(64/3) = 22.
```

If

```text
L_a(X) = prod_{x in E_a}(X-x)
```

is the monic error-root locator, this is equivalently a proper pair-common
locator divisor of degree at least `22`.

If the minimum splice is exact-union, i.e.

```text
supp(Delta) = E_i union E_j union E_k,
```

then `Z` is empty, so `P >= 127`, and some participating pair satisfies

```text
|E_a cap E_b| >= ceil(127/3) = 43.
```

Thus exact-union minimum splices force a proper pair-common locator divisor of
degree at least `43`.  The stronger `43` bound should not be stated for all
minimum defects; silent shells reduce the unconditional minimum-defect
consequence to the `22` bound above.

## Tight all-inside face

There is one sharper all-inside face.  Suppose

```text
U = E_i union E_j union E_k,
|U| = 257,
E_l subset U for all seven retained slopes l.
```

Then the RS shortening to `U` is one-dimensional, generated by

```text
K_U(X) = prod_{h in H \ U}(X-h).
```

Fix two slopes and write

```text
q = (c_2-c_1)/(gamma_2-gamma_1),
p = c_1 - gamma_1 q,
r_l = c_l - (p + gamma_l q).
```

Since all seven errors vanish outside `U`, each `r_l` vanishes outside `U`.
Thus

```text
r_l = lambda_l K_U
```

for scalars `lambda_l`.  For each `x in U`, after dividing by `K_U(x)`, the
condition `x notin E_l` becomes incidence of the point

```text
P_l = (gamma_l, lambda_l)
```

with an affine line depending on `x`.  Each retained slope has exactly
`257 - 128 = 129` no-error coordinates inside `U`, so the seven points have
total incidence

```text
7 * 129 = 903
```

with a multiset of `257` affine lines.

An elementary seven-point incidence bound says that, unless all seven points
are collinear or six of them are collinear, the total incidence is at most
`901`.  Therefore the tight all-inside face gives one of two outcomes:

1. all seven decoded words lie on one affine RS code-line, which is the
   `CCL_149` side; or
2. six of the seven error-root locators have a common divisor of degree at
   least `3`.

We record this second outcome as

```text
SIX_COMMON_GCD_3.
```

It is a proper-subfamily common-GCD branch, not the all-seven
`ALL7_COMMON_GCD_2` branch.

## Residual wall

This audit cuts the overstrong line

```text
TDD257 is paid by CCL_149 or ALL7_COMMON_GCD_2.
```

What survives is narrower:

```text
minimum defect           -> PAIR_COMMON_GCD_22,
exact-union minimum      -> PAIR_COMMON_GCD_43,
tight all-inside minimum -> CCL_149 or SIX_COMMON_GCD_3.
```

These are branch refinements.  They are paid only if the public ledger charges
the corresponding proper-subfamily common-GCD branches.  Otherwise they should
be treated as named residual walls.

The remaining unclosed TDD257 cases are:

```text
OUTSIDE-MCS257,
EXCESS-UNION-TDD258+,
nonzero residual defect-space.
```

Here `EXCESS-UNION-TDD258+` means that the relevant triple union has size at
least `258`; it does not mean that the defect codeword has weight at least
`258`.  A minimum-weight defect can still have a larger triple union through a
silent shell.

## Nonclaims

This note does not close CAP25 v13.  It does not prove the residual numerator
bound.  It does not claim leaderboard progress.  It does not say that
`TDD257` is already paid by the current `CCL_149` or `ALL7_COMMON_GCD_2`
branches.  It only records the first repaired minimum-splice reductions and
the exact residual wall left after them.
